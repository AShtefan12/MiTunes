MiTunes: This media player is to use large text to allow it to be used comfortably with just a wireless mouse from 'across the room'.

Data: The player is to take a file of albums as data. Each album will have a title, an artist name, an artwork filename and a number of tracks. Each track will have a title, an mp3 filename and some lyrics.

Four panes: The media player is to have four panes. Across the top are three panes side-by-side. On the left is a pane to choose an album from a list of albums. In the middle is a pane to choose a track from the tracks on the chosen album. On the right is a pane which allows the chosen track to be played.

Search pane: Underneath the top three panes, a fourth pane stretching the width of the screen provides a text search option. Characters can be entered by clicking simulated keyboard buttons. The keyboard only needs to have 26 letters and 'space' - no shift key, no digits, no punctuation, etc. There should be a facility to delete the last character entered (Back1) and to start the search again from scratch by deleting all the characters entered (Reset).

Search algorithm: The search should be 'live as you type'. After each character is pressed (except the first) a search is made with the current search string. The search is made in the track title and lyrics fields across all albums. It is up to you how the search results are presented to the user - you could re-use one or more of the existing top frames or you could have another (initially invisible/hidden) pane which can be opened/made visible which overlays one or more of the upper panes. The user must be able to select a track from the search results and have it played. The user must be able to 'dismiss' the search results and get back to 'scrolling' through the albums/tracks.

File: MiTunes.zip

You are provided with a folder containing data as a zip file - MiTunes.zip (Note: this is quite large ~300MB). You should download and unzip this file and write your solution at the top level inside this folder. This folder contains descriptions of all the albums together with image files containing album artwork and also audio files for each track. The 'tracks' are copyright-free recordings of birdsong from the BBC.

File: music.js

Inside the (unzipped) MiTunes folder you will find the file music.js. The data containing the description of all the albums in javascript/json format is in the file music.js and can be included in your program by including the line

<script type="text/javascript" src="music.js"></script>

The file music.js begins as follows

var albums = [ .... ]

This will provide you with an initialised global variable called albums. Using this variable you can access each album and each track of each album. Open up the file music.js and study the structure so you can decide when to use array style addressing ( v[...] ) and when to use record style addressing ( v.f ).

Folder: audio

Inside the MiTunes folder there is also a folder (called audio) of audio tracks. The audio folder structure comprises artist folders containing albums, where each album folder contains tracks. Each track is duplicated both in .mp3 and .wav forms (this is to get round some current browser restrictions - see below).

In order to allow the user to play a track, the following HTML5 code should be used, with xxxxx being replaced by the name of the track

<audio controls='controls'>
    <source src='xxxxx.mp3' type='audio/mpeg'>
    <source src='xxxxx.wav' type='audio/wav'>
    Your browser does not support the audio element.
</audio>

If the browser you are using supports HTML5 audio (either .mp3 or .wav) you should be able to click a 'play' button on the line below and hear a Robin singing...

[Tested and working: latest versions of Safari, Firefox, Chrome, Opera, Camino on Mac; Firefox,Chrome on Windows]

[Tested and NOT Working: IE8 on Windows]

This w3Schools page explains the current situation regarding supported audio formats in different browsers in HTML5. Including two alternative audio sources ensures coverage across the major browsers.

Folder: artwork

Also inside the MiTunes folder there is a folder (called artwork) of album artwork. Consult music.js to find the name of the artwork file (say 'artwk1') and then look in artwork for a corresponding jpeg graphics file (e.g. 'artwk1.jpg')
